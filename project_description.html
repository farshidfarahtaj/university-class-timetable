University Timetable Management System - Project Overview

1. Project Purpose

This is a comprehensive web-based university timetable management system designed to streamline the scheduling and management of university classes. The system serves two main user groups: administrators and students, each with specific functionalities tailored to their needs.

2. Technical Architecture

The project is built using:

  • Backend: PHP for server-side processing
  • Database: MySQL for data storage
  • Frontend: HTML, CSS, and JavaScript
  • Security: Session-based authentication with password hashing

3. Key Features and Sections

A. User Management System

1. Authentication System

  • Secure login system (login.php)
  • Password reset functionality (reset_password.php)
  • User registration (register.php)
  • Session management and logout (logout.php)

2. User Profiles

  • Student profiles (user_profile.php)
  • Admin profiles (modify_admins.php)
  • User modification capabilities (modify_user.php)

B. Administrative Features

1. Dashboard

  • Admin dashboard (admin_dashboard.php)
  • Student dashboard (student_dashboard.php)
  • User management interface (manage_users.php)

2. Resource Management

  • Course management (manage_courses.php, edit_course.php)
  • Subject management (manage_subjects.php, edit_subject.php)
  • Room management (manage_rooms.php, edit_room.php)

3. Timetable Management

  • Timetable creation and editing (manage_timetables.php)
  • Timetable modification (edit_timetable.php)
  • Timetable viewing (view_timetable.php)

C. Student Features

1. Personal Dashboard

  • Personalized timetable view
  • Course schedule access
  • Profile management

2. Timetable Access

  • View schedules based on their major
  • Filter and search capabilities

D. UI/UX Components

1. Navigation

  • Left sidebar navigation (LeftSidebar.php)
  • Responsive design elements

2. Styling

  • Main stylesheet (style.css)
  • Admin-specific styling (admin-style.css)
  • Sidebar styling (left-sidebar.css)

4. Security Features

  • Secure password handling
  • Session-based authentication
  • User role management
  • Protected routes and resources

5. Database Structure

The system uses a well-structured MySQL database (database.sql) that includes:

  • User tables
  • Course and subject tables
  • Room management tables
  • Timetable scheduling tables

6. Modern Features

  • Responsive design
  • Modern UI components
  • Interactive elements using JavaScript
  • Real-time updates and modifications

7. Project Organization

The project follows a clear file structure:

  • Core functionality files (PHP)
  • Style sheets (CSS)
  • Client-side scripts (JavaScript)
  • Database configuration and structure
  • Documentation files

8. Future Development

The project includes modernization guides (admin_modernization_guide.md) indicating ongoing development and improvements to the system.

9. Educational Value

This system demonstrates a practical application of web programming concepts, including:

  • Database management
  • User authentication
  • CRUD operations
  • Frontend-backend integration
  • Security best practices
  • Responsive design principles
